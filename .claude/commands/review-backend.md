현재 git 변경사항을 **백엔드 관점**에서 검토하고 다음 항목들을 확인해주세요:

## 검토 항목

### 1. 코드 품질
- 함수/클래스 단일 책임 원칙 (SRP)
- 네이밍 컨벤션 일관성
- 코드 중복 및 DRY 원칙
- 적절한 추상화 레벨

### 2. API 설계
- RESTful/GraphQL 규칙 준수
- 적절한 HTTP 메서드 사용 (GET, POST, PUT, DELETE)
- 상태 코드 정확성 (200, 201, 400, 401, 404, 500 등)
- API 버저닝 전략
- 페이지네이션 구현

### 3. 보안
- **SQL Injection** 방지
  - Prepared statements/Parameterized queries 사용
  - ORM 사용 시에도 raw query 주의
- **인증/인가**
  - JWT 토큰 검증
  - 권한 체크 (RBAC, ABAC)
  - 세션 관리
- **민감 정보 보호**
  - 비밀번호 해싱 (bcrypt, argon2)
  - 환경 변수 사용 (.env)
  - API 키, 시크릿 노출 방지
- **입력 유효성 검사**
  - 서버 사이드 validation 필수
  - 스키마 validation (Joi, Zod 등)
- **Rate Limiting**
  - DDoS 방지
  - Brute force 공격 방지
- **CORS** 설정 적절성

### 4. 데이터베이스
- 쿼리 최적화 (N+1 문제)
- 인덱스 사용 적절성
- 트랜잭션 처리
- 데이터 무결성 제약조건
- 마이그레이션 전략

### 5. 에러 핸들링
- try-catch 적절한 사용
- 에러 로깅 (stack trace, 컨텍스트 정보)
- 사용자 친화적 에러 메시지 (내부 정보 노출 방지)
- 에러 모니터링 (Sentry 등)

### 6. 성능
- 데이터베이스 쿼리 성능
- 캐싱 전략 (Redis, Memcached)
- 비동기 처리 (큐, 백그라운드 작업)
- 연결 풀링

### 7. 테스트
- 단위 테스트 커버리지
- 통합 테스트
- API 엔드포인트 테스트
- 엣지 케이스 처리

각 항목에 대해 구체적인 피드백을 제공하고, 개선이 필요한 부분은 코드 예시와 함께 제안해주세요.
