현재 git 변경사항을 검토해주세요.

## 자동 감지 및 리뷰

다음 단계로 진행하세요:

1. **변경된 파일 확인**: `git status`와 `git diff`로 변경된 파일 목록 확인

2. **파일 타입 분류**: 변경된 파일들을 분석하여:
   - **프론트엔드**: React 컴포넌트(.tsx, .jsx), 스타일(.css, .scss), 프론트엔드 utils
   - **백엔드**: API 라우터, 컨트롤러, 서비스, 데이터베이스 모델, 미들웨어
   - **풀스택**: 둘 다 포함된 경우

3. **적절한 리뷰 수행**:

   ### 프론트엔드 파일인 경우:
   - React/프론트엔드 특화 사항 (리렌더링, 상태 관리)
   - XSS 취약점
   - 클라이언트 성능 (번들 사이즈, lazy loading)
   - 접근성 (a11y)
   - null/undefined 체크

   ### 백엔드 파일인 경우:
   - SQL Injection 방지
   - 인증/인가 로직
   - 서버 사이드 validation
   - 데이터베이스 쿼리 최적화
   - Rate limiting
   - 에러 핸들링 및 로깅

   ### 풀스택인 경우:
   - 프론트엔드와 백엔드 항목 모두 체크
   - API 연동 부분 특히 주의

4. **공통 검토 항목**:
   - 코드 품질 (가독성, 네이밍, DRY)
   - 보안 (민감 정보 노출, 환경 변수)
   - 테스트 커버리지
   - 타입 안전성 (TypeScript)

각 항목에 대해 구체적인 피드백을 제공하고, 개선이 필요한 부분은 코드 예시와 함께 제안해주세요.

---

- `/review-backend` - 백엔드 전용 리뷰
